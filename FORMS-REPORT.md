# Отчёт тестирования форм — MELARDO WARFACE

**Дата:** 8 марта 2025  
**Метод:** E2E тесты Playwright (`e2e/forms.spec.ts`) + анализ кода

---

## 1. Регистрация (/register)

| Проверка | Результат |
|----------|------------|
| Заполнение тестовыми данными | ✓ |
| Валидация (пустые поля) | ✓ HTML5 `required` |
| Валидация (неверный формат ника) | ✓ `pattern`, `minLength`, `maxLength` (3–16 символов, только русские буквы, цифры, _ - .) |
| Отправка формы | — Не выполнялась (избежание создания тестовых аккаунтов) |

**Серверная валидация:** `signUp` в `app/actions/auth.ts` — проверка ника, уникальность, Supabase `signUp`.

---

## 2. Вход (/login)

| Проверка | Результат |
|----------|------------|
| Неверные данные | ✓ Ошибка или остаётся на /login |
| Верные данные | — Требуется реальный аккаунт |

**API:** Supabase `signInWithPassword`. При успехе — редирект на `/profile`.

---

## 3. Создание команды (/teams/create)

| Проверка | Результат |
|----------|------------|
| Доступ без авторизации | ✓ Редирект на /login |
| Заполнение (название, город, описание) | ✓ |
| Валидация | ✓ react-hook-form: название 3–20 символов, только русские буквы/цифры/пробелы, фильтр нецензурных слов |
| Отправка | ✓ API `/api/team/create` — создаёт команду и добавляет капитана |

**Ограничение:** один пользователь может быть капитаном только одной команды.

---

## 4. Создание вызова (/matches/create)

| Проверка | Результат |
|----------|------------|
| Доступ без авторизации | ✓ Редирект на /login |
| Селект команд | ✓ **Исправлено:** добавлен API `/api/team/my-teams` и загрузка команд в select |
| Режим, дата, время, комментарий | ✓ |
| Валидация | ✓ Выбор команды обязателен (клиент + API) |

**Исправление:** селект «Ваша команда» раньше был пустым; теперь подгружаются команды пользователя через `/api/team/my-teams`.

---

## 5. Запрос турнира (/tournaments/request)

| Проверка | Результат |
|----------|------------|
| Доступ без авторизации | ✓ Редирект на /login |
| Заполнение полей | ✓ Название, режим (5x5/8x8), формат, min/max команд, дата, комментарий, чекбокс fair play |
| Валидация | ✓ Название, min/max (2–64), fair play обязателен |
| Отправка | ✓ API `/api/tournament/request` — запись в `tournament_requests` |

---

## 6. Подтверждение матча (/matches/confirm/[id])

| Проверка | Результат |
|----------|------------|
| Доступ | Страница матча; только капитаны команд могут подтверждать |
| Поля | Счёт команды 1, счёт команды 2, скриншот (JPG/PNG), чекбокс подтверждения |
| Валидация | ✓ API проверяет счёт, формат файла, чекбокс |
| Отправка | ✓ API `/api/match/confirm` — загрузка в Storage `match-screenshots`, запись в `match_confirmations` |

**Файлы:** сохраняются в Supabase Storage, bucket `match-screenshots`.

---

## 7. Обратная связь (/support)

| Проверка | Результат |
|----------|------------|
| Заполнение | ✓ Имя, email, тема, сообщение |
| Прикрепление файла | ✓ Опционально, до 5 МБ; для загрузки нужна авторизация |
| Валидация | ✓ Клиент + API: email, тема из списка, сообщение |
| Отправка | ✓ API `/api/support/send` — запись в `support_messages` |

**Примечание:** в E2E-тесте результат отправки «не определён» — возможно, из‑за отсутствия Supabase в тестовом окружении или таймаута. При корректной настройке Supabase форма должна работать.

---

## 8. Жалоба (модальное окно)

| Проверка | Результат |
|----------|------------|
| Где отображается | Страницы матча (`/matches/[id]`), подтверждения (`/matches/confirm/[id]`), профиля игрока (`/players/[id]`), команды (`/teams/[id]`) |
| Причина | Select: Неверный результат, Читы, Оскорбления, Подставной игрок, Другое |
| Комментарий | Текстовое поле |
| Скриншот/видео | Опционально (PNG, JPG, WebP, MP4, WebM) |
| Отправка | ✓ API `/api/complaint/create` — запись в `complaints`, файлы в Storage `complaints` |

**Требуется авторизация.**
