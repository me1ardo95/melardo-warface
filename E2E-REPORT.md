## 5. Ошибки в консоли

Зафиксирована **одна** ошибка:

- **Страница:** `http://localhost:3000/support`
- **Тип:** предупреждение о несовпадении гидрации (hydration mismatch).
- **Суть:** на клиенте у полей формы (input/textarea/file) появляется `style={{ caret-color: "transparent" }}`, которого нет в серверном HTML, из-за чего React сообщает о расхождении.

**Рекомендация:** добавлять `caret-color: transparent` только на клиенте (например, в `useEffect` после монтирования) или не задавать этот стиль для полей, рендерящихся и на сервере, и на клиенте в одном и том же виде.

## 6. API и сетевые запросы

В отчёте тестов перечислены «неудачные» запросы (`net::ERR_ABORTED`). По типам это:

- Чанки Next.js/Turbopack (`_next/static/chunks/...`) — обрыв при навигации или HMR.
- RSC-запросы (`?_rsc=...`) — отмена при переходе на другую страницу.
- Шрифты (`woff2`), HMR-клиент.

Такое поведение в режиме **dev** нормально и не указывает на сбой API приложения. Отдельных падений API (например, Supabase или своих маршрутов) в отчёте не зафиксировано.

## 8. Рекомендации по исправлению

1. **Страница `/rules` в E2E**  
   В `e2e/full-platform.spec.ts` для `/rules` сейчас ожидается 404. Нужно изменить ожидание: при переходе на `/rules` ожидать статус 200 и, при желании, наличие ключевого текста с страницы правил.

2. **Добавить проверку `/guide`**  
   В блок «Публичные страницы» добавить маршрут `/guide` и проверять загрузку (200).

3. **Кнопка «Случайный соперник»**  
   Уточнить селектор: например, `page.getByRole('button').filter({ hasText: 'Случайный соперник' })` или проверять по части текста без эмодзи, при необходимости с `await expect(btn).toBeVisible({ timeout: 5000 })` после перехода на `/matches`.

4. **Гидрация на `/support`**  
   Убрать расхождение по `caret-color`: не применять этот стиль к полям формы при первом рендере на сервере или применять его только после монтирования на клиенте.

5. **Предупреждение Next.js**  
   В логе при запуске есть предупреждение о конвенции `middleware` (рекомендация перейти на `proxy`). Имеет смысл ознакомиться с документацией Next.js и по возможности перейти на рекомендуемый подход.
